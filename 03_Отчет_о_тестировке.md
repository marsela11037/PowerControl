# Отчет о тестировке

## 1. Объект тестирования
- Приложение: `Power Control`
- Основной модуль: `power_app.py`
- Версия: текущая рабочая версия репозитория
- Дата: 2026-04-16

## 2. Подход
- Статическая проверка Python-кода.
- Проверка импортов и синтаксической корректности.
- Функциональные сценарии сформированы как чек-лист для ручного прогона (GUI + действия питания требуют безопасной среды).

## 3. Выполненные проверки (автоматически)
1. Команда: `py -3 -m py_compile power_app.py build_exe.py`
2. Ожидаемый результат: отсутствие синтаксических ошибок.
3. Фактический результат: **пройдено**, код возврата `0`, ошибки компиляции не обнаружены.

## 4. Чек-лист ручного тестирования
### 4.1 Запуск и интерфейс
- [ ] Приложение запускается без ошибок.
- [ ] Сервер стартует автоматически.
- [ ] Отображаются IP, порт, токен и URL.
- [ ] QR-код генерируется и открывается в отдельном окне.

### 4.2 Настройки
- [ ] Изменение порта в диапазоне `1024-65535` сохраняется.
- [ ] Изменение токена сохраняется.
- [ ] После перезапуска приложения токен и порт восстанавливаются из конфига.

### 4.3 Сервер и API
- [ ] `GET /health` возвращает `200 OK`.
- [ ] `GET /shutdown` с неверным токеном возвращает `403`.
- [ ] `GET /sleep` с неверным токеном возвращает `403`.
- [ ] `POST /api/shutdown` с валидным токеном возвращает `200`.
- [ ] `POST /api/sleep` с валидным токеном возвращает `200`.
- [ ] `POST /api/wake` с валидным `pc_id` возвращает `200`.

### 4.4 Трей и автозапуск
- [ ] Закрытие окна предлагает свернуть в трей.
- [ ] Из трея работает возврат в окно и выход.
- [ ] Переключатель "Запускать с Windows" корректно добавляет/удаляет запись из реестра.

## 5. Риски и ограничения
- Команды сна/выключения нельзя безопасно гонять в рабочей среде без плана отката.
- Wake-on-LAN зависит от BIOS/UEFI и сетевой инфраструктуры.
- При занятом порте сервер не стартует (должна показываться ошибка).
