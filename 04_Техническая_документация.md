# Техническая документация

## 1. Архитектура
Приложение построено как монолитный десктоп-модуль `power_app.py`, который объединяет:
- GUI на `tkinter`;
- HTTP-сервер на `http.server.HTTPServer`;
- обработчики команд управления питанием;
- системный трей (`pystray`);
- генерацию QR (`qrcode`, `Pillow`);
- хранение пользовательской конфигурации в JSON-файле.

## 2. Основные компоненты
### 2.1 Конфигурация
- `PORT` и `SECRET_TOKEN` читаются из env:
  - `POWER_PORT` (по умолчанию `8765`);
  - `POWER_TOKEN` (по умолчанию `marsela`).
- `WOL_COMPUTERS` хранит профиль(и) ПК для WoL.

### 2.2 HTTP слой (`PowerHandler`)
- `GET /` -> отдает `templates/index.html`.
- `GET /static/*` -> отдает статические ресурсы.
- `GET /health` -> health-check.
- `GET /shutdown`, `GET /sleep` -> команды при валидном query-токене.
- `POST /api/shutdown`, `/api/sleep`, `/api/wake` -> JSON API с токеном в body.

### 2.3 GUI слой (`PowerControlApp`)
- Показывает IP/порт, URL и токен.
- Позволяет редактировать порт и токен.
- Запускает/останавливает HTTP-сервер.
- Управляет автозапуском через реестр Windows.
- Поддерживает сворачивание в системный трей.

## 3. Поток выполнения
1. Старт приложения (`__main__`).
2. Загрузка конфигурации из `%USERPROFILE%\\.power_control_config.json`.
3. Построение GUI.
4. Автоматический запуск HTTP-сервера (`root.after(500, self.toggle_server)`).
5. Обработка входящих запросов телефона.

## 4. Безопасность
- Контроль доступа основан на токене.
- Токен передается:
  - в query-параметре (GET);
  - в JSON body (POST).
- Рекомендация: использовать длинный токен и ограничивать доступ локальной сетью.

## 5. Сборка и деплой
- Для сборки используется `PyInstaller` (`build_simple.bat`/`build_exe.py`).
- В EXE включаются каталоги `templates` и `static`.
- Результат сборки: `dist/PowerControl.exe`.

## 6. Зависимости
- `pystray`, `Pillow` - трей и графика.
- `qrcode` - QR-код URL.
- `pyinstaller` - упаковка в EXE.
- `pyperclip` - вспомогательная библиотека (в текущей реализации почти не используется).

## 7. Технический долг и улучшения
- Разделить `power_app.py` на модули:
  - `server.py`, `gui.py`, `config.py`, `wol.py`.
- Внедрить structured logging.
- Добавить тестовый контур (unit/integration) для API.
- Добавить rate-limit и блокировку после серии неверных токенов.
